INICIO

Comandos Git

Inicializando un repositorio en un directorio existente

        $ git init

Clonando un repositorio exitente usando el protocolo https://

        Clonando un repositorio con el nombre de directorio por defecto
        $ git clone [url]
        $ git clone https://github.com/libgit2/libgit2

        Clonando el repositorio con un nombre de directorio diferente
        $ git clone [url] [projetcA]
        $ git clone https://github.com/libgit2/libgit2 myLibGit2

Revisando el Estado de tus Archivos

        $ git status
        On branch master
        nothing to commit, working directory clean

Rastrear Archivos Nuevos y Preparar Archivos Modificados "Staged (Preparado)" 

        Todos los archivos que tiene un "c" despues del punto
        $ git add *.c

        Solo un archivo
        $ git add index.html

        Todos los cambios
        $ git add .

Confirmar tus Cambios "Commit (Confirmado)"

        Al hacerlo, arrancará el editor de tu preferencia
        $ git commit

        Escribir el mensaje de confirmación directamente en el comando commit utilizando la opción -m
        $ git commit -m 'initial project version'

Eliminar Archivos

Si simplemente eliminas el archivo de tu directorio de trabajo, aparecerá en la sección “Changes not staged for commit” (esto es, sin preparar) en la salida de git status: 

        $ rm PROJECTS.md

        $ git status
        On branch master
        Your branch is up-to-date with 'origin/master'.
        Changes not staged for commit:
          (use "git add/rm file..." to update what will be committed)
          (use "git checkout -- file..." to discard changes in working directory)
          deleted: PROJECTS.md
        no changes added to commit (use "git add" and/or "git commit -a")

Ahora, si ejecutas git rm, entonces se prepara la eliminación del archivo: 

        $ git rm PROJECTS.md
        rm 'PROJECTS.md'

        $ git status
        On branch master
        Changes to be committed:
          (use "git reset HEAD file..." to unstage)
          deleted: PROJECTS.md

Cambiar el Nombre de los Archivos

        $ git mv file_from file_to

        Su equivalencia o hecho de otro modo
        $ git mv README.md README

        $ git status
        On branch master
        Changes to be committed:
          (use "git reset HEAD file..." to unstage)
          renamed: README.md -> README

Ver el Historial de Confirmaciones

        $ git log

        commit ca82a6dff817ec66f44342007202690a93763949
        Author: Scott Chacon schacon@gee-mail.com
        Date: Mon Mar 17 21:52:11 2008 -0700
          changed the version number

Mostrar las diferencias introducidas en cada confirmación

        $ git log -p

Mostrar únicamente las 2 últimas entradas del historial:

        $ git log -2

Ver algunas estadísticas de cada confirmación

        $ git log --stat

Format te permite especificar tu propio formato

        $ git log --pretty=format:"%h - %an, %ar : %s"

        ca82a6d - Scott Chacon, 6 years ago : changed the version number
        085bb3b - Scott Chacon, 6 years ago : removed unnecessary test
        a11bef0 - Scott Chacon, 6 years ago : first commit

Deshacer un archivo Confirmado

        $ git commit --amend

Deshacer un Archivo Preparado

        Primera forma
        $ git restore --staged "archivo..."

        Segunda forma
        $ git reset HEAD "archivo..."

Deshacer un Archivo Modificado

        Primera forma
        $ git restore "archivo..."

        Segunda forma
        $ git checkout -- "archivo..."
